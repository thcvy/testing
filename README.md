# FormulaVRviewer

EEEE2076 Software Engineering & VR Project &mdash; **Group Task**.

A Qt + VTK desktop application that loads CAD models in STL format and
displays them both in a regular OpenGL window *and* in a VR headset
through OpenVR / SteamVR.

## At a glance

|              |                                                    |
|--------------|----------------------------------------------------|
| GUI          | Qt 6 (Widgets), `QVTKOpenGLNativeWidget` viewport |
| 3D engine    | VTK 9 (with the OpenVR module)                     |
| VR runtime   | OpenVR / SteamVR                                   |
| Build system | CMake 3.19+                                        |
| Installer    | NSIS (via CPack)                                   |
| Documentation| Doxygen + GitHub Pages                             |

## Features

- Load individual STL files or a whole folder of them
- Tree-view scene graph with add / remove / properties / right-click context menu
- Two combinable filters per part (Shrink, Clip)
- Independent name / colour / visibility per part
- Light editor and background-colour editor
- Dark mode toggle
- VR scene with adjustable per-axis rotation, reset view, environment toggle
- Live colour / visibility / new-actor updates while VR is running
- Stable VR start / stop across many cycles

## Building

You need:

- A recent CMake (>=3.19)
- Qt 6.5 or later (Widgets module)
- VTK 9 built with the **OpenVR** module enabled
- OpenVR runtime (Steam + SteamVR) installed for the VR side
- A C++17 compiler (MSVC 2019+, GCC 11+, Clang 13+)
- *(Optional)* Doxygen + Graphviz for the documentation target
- *(Optional)* NSIS for building the Windows installer

Then:

```bash
cmake -S . -B build
cmake --build build --config Release
```

Run from the build tree:

```bash
./build/FormulaVRviewer        # Linux / macOS
build\Release\FormulaVRviewer  # Windows
```

## Installing on Windows

The project ships a CPack/NSIS installer recipe.  After building:

```bash
cmake --build build --config Release --target package
```

This produces a self-contained `FormulaVRviewer-1.0.0-win64.exe` that:

- Bundles the Visual C++ runtime (via `InstallRequiredSystemLibraries`)
- Bundles only the Qt DLLs the app actually imports (via `windeployqt`)
- Bundles the VTK DLLs (resolved automatically from the link step)
- Bundles the OpenVR runtime DLLs
- Drops the `vrbindings/` action JSONs next to the exe
- Adds a Start Menu shortcut and a working uninstaller

Inside Visual Studio, the same thing is reachable as the **`PACKAGE`**
target on the solution.  The standard **`INSTALL`** target lays out the
self-contained directory tree without wrapping it into an installer.

## Documentation

API documentation is generated by Doxygen.  Locally:

```bash
cmake --build build --target Doxygen
xdg-open build/docs/html/index.html       # or just open it in a browser
```

The same docs are auto-published to GitHub Pages on every push to
`main` &mdash; see `.github/workflows/docs.yml`.
